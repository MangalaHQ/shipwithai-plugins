## VIII. Open-Source Integration — Curate, Don't Fork

### Principle: Don't fork-and-sell. Do curate-and-bundle.

| Open-source tool | How to integrate | Value you add |
|---|---|---|
| Supermemory | Skill "setup memory for Claude Code" | Optimized config, integration guide |
| Better Auth (free) | Skill "auth from zero to production" | Complete workflow, edge cases, migration |
| Drizzle ORM (free) | Skill "database schema design" | Best practices, common patterns, auto-migration |
| Resend (free tier) | Skill "email setup" | Templates, transactional patterns |
| Stripe SDK (free) | Skill "payment integration" | Webhook verification, subscription flows |

**You don't sell the tool — you sell expertise packaged in format Claude understands best.**

Anyone can use Better Auth for free, but Claude takes 35,000 tokens to figure out how to use it. Your skill reduces it to 8,000 tokens → save tokens = save money for user.

### Contribute Back

- Contribute code to open-source projects you use
- Build reputation in community
- This is also a marketing channel — maintainers will mention you
